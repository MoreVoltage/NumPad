## Reusable recipes / gotchas (this machine)
- **Fast model TDD loop:** scratch SPM at `/tmp/LayoutKit` (`swift test`, sub-second). Write/verify pure logic there, then copy into `NumPad/Libraries/Customization/` + `NumPadTests/` (change `@testable import LayoutKit` → `NumPad`).
- **Add files to Xcode targets:** the `xcodeproj` gem isn't in Homebrew ruby by default — `gem install xcodeproj` (done). Wire via a Ruby script (pattern in `/tmp/wire.rb`): anchor groups off existing files (`SharedExtensions.swift` → Customization group app+ext; `Item.swift` → Keyboard group; `NumPadTests.swift` → test group), `add_file_references` per target. App-only files → `[app]`; extension-only → `[kb]`.
- **`Color` is shadowed (app target):** `SharedExtensions.swift` declares `typealias Color = UIColor.Custom` at module scope, so in any **app-target SwiftUI** file a bare `Color(...)`/`Color.clear`/`Color.accentColor` resolves to `UIColor.Custom` and fails to compile (`requires that 'UIColor.Custom' conform to 'ShapeStyle'`). Fully-qualify as `SwiftUI.Color`. (`Color` is the only colliding alias — other SwiftUI types are safe.)
- **Run the real suite:** boot a sim first, then
  `xcodebuild test -workspace NumPad.xcworkspace -scheme NumPad -destination 'platform=iOS Simulator,id=<UDID>' -parallel-testing-enabled NO -derivedDataPath /Volumes/DevVault/DerivedData_numpad`
  (the `-parallel-testing-enabled NO` + pre-booted sim avoids the sim-clone hang; DerivedData on DevVault avoids main-disk pressure).
- **Run on a device (dev build, verified working):** `xcodebuild -workspace NumPad.xcworkspace -scheme NumPad -configuration Debug -destination 'platform=iOS,id=<UDID>' -derivedDataPath /Volumes/DevVault/DerivedData_numpad -allowProvisioningUpdates build` (automatic signing, team `NNRNHY2N8B`; app+extension provision on the fly), then `xcrun devicectl device install app --device <coredevice-UUID> .../Build/Products/Debug-iphoneos/NumPad.app` and `xcrun devicectl device process launch --device <coredevice-UUID> com.morevoltage.NumPad`. Devices: **iPhone 17** (27.0) UDID `00008150-001E0CEE3C41401C` / coredevice `25422028-6E89-549C-84E4-98E7CFFC4E08`; **iPhone 16 Pro** (26.5) UDID `00008140-0008198E3862201C`. The CLI can't screenshot/drive a physical device — interactive UI checks are manual. To bypass gating while testing, use the **DEBUG-only Debug section** in the Store screen to simulate entitlement.
