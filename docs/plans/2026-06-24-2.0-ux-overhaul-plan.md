# NumPad 2.0 UX Overhaul — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship the five approved UX fixes on `feat/2.0-ux-overhaul`: packs 12→5, a reframed notification ask, richer/clearer snippets, a custom-keys preview screen, and a single springboard-style custom keyboard.

**Architecture:** UIKit MVC app + keyboard extension sharing an app group; new editor screens are SwiftUI islands (UIHostingController) per the existing `Customization/Editor` pattern. Pure logic lives in shared, unit-tested types (`Monetization`, `PackKeys`, `Snippet`, `DateTimeTokens`, `KeyboardLayout`); view controllers are build/sim-verified (the suite never unit-tests VCs).

**Tech Stack:** Swift, UIKit, SwiftUI (iOS 16 island APIs — `ObservableObject`, not `@Observable`), CocoaPods, StoreKit 2, Firebase Remote Config, XCTest.

**Source of truth for decisions:** `docs/plans/2026-06-24-2.0-ux-overhaul-design.md`.

## Conventions (read once)

- **Immutable edits** — every `KeyboardLayout` op returns a new copy (see `LayoutEditing.swift`). Keep that style.
- **Cross-process sync** — after any change the keyboard must see, write to `UserDefaults.group` then call `SettingsSync.post()`.
- **Test command** (boot the sim once first: `xcrun simctl boot "iPhone 16 Pro" 2>/dev/null || true`):
  ```bash
  xcodebuild test -workspace NumPad.xcworkspace -scheme NumPad \
    -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
    -derivedDataPath /Volumes/DevVault/DerivedData_numpad \
    -parallel-testing-enabled NO \
    -only-testing:NumPadTests/<Class>/<method>
  ```
  Full build: same without `test`/`-only-testing` → `xcodebuild build -workspace … -scheme NumPad …`.
- **Adding a NEW .swift file to a target** (sources → `NumPad`/`Keyboard`; tests → `NumPadTests`) requires editing `NumPad.xcodeproj/project.pbxproj`. Homebrew ruby lacks it: `gem install xcodeproj` first, then:
  ```ruby
  # scripts/add_file.rb  —  ruby scripts/add_file.rb <relpath.swift> <TargetName>
  require 'xcodeproj'
  proj = Xcodeproj::Project.open('NumPad.xcodeproj')
  path, target_name = ARGV[0], ARGV[1]
  target = proj.targets.find { |t| t.name == target_name }
  group  = proj.main_group # adjust to the matching group if needed
  ref = group.find_file_by_path(path) || group.new_file(path)
  target.add_file_references([ref]) unless target.source_build_phase.files_references.include?(ref)
  proj.save
  ```
  Prefer adding tests to **existing** files where possible to avoid this.
- **After each phase:** run `graphify update .` (AST-only, keeps the graph current per CLAUDE.md).
- **Commit** after every green task. Conventional commits, no attribution (global setting).

```swift
extension Snippet {
    static let tokenMap: [(token: String, dt: String?)] = [
        ("{date}","date"), ("{time}","time"), ("{datetime}","datetime"),
        ("{day}","weekday"), ("{month}","month"), ("{year}","year"),
        ("{iso}","iso"), ("{unix}","unix"), ("{clipboard}", nil)]

    static func expand(_ text: String, now: Date = Date(), locale: Locale = .current,
                       clipboard: () -> String? = { ClipboardHistoryManager.shared.entries.first?.text }) -> String {
        var out = text
        for (token, dt) in tokenMap where out.contains(token) {
            let value = dt.flatMap { DateTimeTokens.value(for: $0, now: now, locale: locale) } ?? clipboard() ?? ""
            out = out.replacingOccurrences(of: token, with: value)
        }
        return out
    }
    func expandedText(now: Date = Date()) -> String { Snippet.expand(text, now: now) }
}
```
