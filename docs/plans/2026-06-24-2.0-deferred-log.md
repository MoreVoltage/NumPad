## Phase 3 — GA promotion notes (2026-06-24)
- Promoted to GA (now `UserPrefs`, default **ON** for everyone; toggle in Store → Settings):
  **Inline Calculator, Cursor Controls, Smart Pack Defaulting, Result Tape.**
- ⚠️ **Inline Calculator was promoted although not in the explicit list** — the Result Tape is fed
  *only* by calculator results, so the tape is meaningless without it. Say if you'd rather keep the
  calculator off (then the tape should be cut too).
- Defaulting Inline Calculator ON changes the **"="** key to evaluate the trailing expression (it
  falls back to inserting a literal "=" when the text isn't a valid expression). Confirm that's desired.
- NOT promoted (still experimental, DEBUG/TestFlight-only): Conversion Overlay, Locale-Aware
  Separators, Save Snippet From Keyboard, Fast Delete. Easy to promote later if wanted.
- Dead code: removed the 4 promoted getters/stored props from `FeatureFlags`; the old `ff*` Constants
  keys for them are now unused (harmless) — clean up in a later pass.

## Phase 3 — iCloud sync notes (2026-06-24)
- Implemented as `CloudSync`: app-group ↔ `NSUbiquitousKeyValueStore` mirror, **Pro-gated** + user
  opt-in (Store → Settings → iCloud Sync; default OFF). Syncs snippets, custom pack/slots, custom
  layouts + active id, theme, height preset. Push on background; pull on foreground + external change.
- ⚠️ **Clipboard is intentionally EXCLUDED despite "option 2".** Clipboard history lives in the
  **keychain** (it can hold passwords / 2FA / card numbers); copying it into plaintext iCloud KVS
  would regress that security decision. Proper path = iCloud **Keychain** (`kSecAttrSynchronizable`),
  a separate effort — **DEFERRED**. Confirm you accept clipboard not syncing for now.
- ⚠️ **Owner-gated capability:** the iCloud **Key-Value storage** capability must be enabled for the
  app id in the Apple Developer portal / Xcode signing. The entitlement is declared in
  `NumPad.entitlements`, but device/archive builds need it provisioned. Until then, KVS calls no-op.
- ⚠️ **Owner-gated verification:** real cross-device sync needs two iCloud-signed devices — NOT verified.
- Conflict strategy is **whole-value last-writer-wins per key** (KVS default), not a semantic union
  merge — concurrent edits to a collection on two devices can drop the older side. Union-merge is a
  refinement — **DEFERRED**.
