# NumPad 2.0 — UX Overhaul Design

**Date:** 2026-06-24
**Branch:** `feat/2.0` (implementation to branch from here)
**Status:** Design approved (brainstorming complete). Not yet implemented.
**Companion docs:** `2026-06-24-2.0-monetization-and-scope-design.md`, `2026-06-24-2.0-deferred-log.md`

This overhaul addresses five UX problems found in the `feat/2.0` build before shipping: an
App-Review-risky notification ask, an over-extensive pack list (12 options / 9 IAPs), an
under-explained snippets feature, an "atrocious" custom-keys setup, and an "awful" custom-keyboard
editor lacking real direct manipulation.

---

## Decisions (approved)

| Area | Decision |
|---|---|
| **Packs** | Curate **12 → 5**. Keep à-la-carte for **new** users; **grandfather** existing owners (keep everything). **Pro** repositions around the *custom* features and still bundles all packs + themes + sync. |
| **Tokens** | Invest in **richer snippet tokens only** — do not extend the token system into packs or custom keys. |
| **Notifications** | **Reframe** the permission as product-updates; keep early-bird as one such update; add a contextual pre-prompt. |
| **Custom keys gating** | Build-your-own **pack row = Pro**; **basic right-edge slot remapping stays free**. |
| **Custom keyboard** | **One** keyboard, open the **editor directly** (skip the list), **springboard-style** drag-and-drop (hand-rolled). Retain multi-layout code, unsurfaced. |

## Section 1 — Packs: 12 → 5

### The 5 (merge map)

| Keep | Absorbs | Notes |
|---|---|---|
| **Math** *(free)* | Math2 (as a layout variant) | Default extra row; free with the paid app |
| **Symbols & Science** | Symbols + Scientific | `π e √ ^ ² ³` + common symbols |
| **Finance** | Business + International currency | `$ € £ ¥ % ‰`; **keeps the live `numpad.pack.finance` product ID** so grandfathering is automatic |
| **Programmer** | Programmer+ | hex, bitwise, `{ } != == && \|\| ->` |
| **Date & Time** *(computed)* | — | Live-value row stays first-class |

**Dropped:** Math2 (folded), Scientific (→Symbols & Science), Business + International (→Finance),
Programmer+ (→Programmer), and **Units & Conversion** (dropped entirely — weakest pack; real
conversion needs live FX, already cut in the deferred log).

### Gating

- **Math** — free with the paid app.
- **Symbols & Science / Finance / Programmer / Date & Time** — **$1.99 à-la-carte** each, *or* all included in **Pro**.
- **Pro ($11.99)** — all 4 premium packs + custom keys + custom keyboard + premium themes + iCloud sync.
- **Grandfathered** (pre-2.0 users + existing Finance/Pro buyers) — keep everything; the old
  `numpad.pack.finance` purchase maps onto the merged Finance pack.

### Impact

- ASC catalog: **11 IAPs → 6** (4 pack SKUs + 2 Pro SKUs: `numpad.pro.lifetime` + `numpad.pro.lifetime.earlybird`).
- Picker: **12 rows → 6** (None + 5).

### Implementation touchpoints

- `Keyboard.swift` — `KeyboardType` enum + `KeyboardType.packs`; collapse merged cases (retain
  dropped cases as deprecated aliases for back-compat / grandfathering).
- `Keyboard/Libraries/Item.swift` — `PackKeys` rows for the merged packs (rebuild the 4 premium rows).
- `SharedExtensions.swift` — `ProductCatalog` (drop the removed pack SKUs), `RemoteConfigManager.enabledPacks`
  (new default list), `Monetization` gating + the grandfathering map for Finance.
- `PacksViewController.swift` — render 6 rows.
- `FeaturesGuideViewController.swift` — update the Packs section copy.

## Section 2 — Snippets: explain it, make tokens discoverable

**Problems:** a single tiny footer tip is the only explanation; the composer is a cramped inline
table cell; `{date}`/`{time}` are invisible unless already known.

**Redesign:**
1. **Empty-state / header** stating the model: *"Snippets are reusable text you insert from the
   keyboard — long-press the **.** key. Add tokens to drop in live values."*
2. **Token-picker chips in the composer** (Date · Time · Date+Time · Day · Month · Year · ISO · Unix ·
   Clipboard) that insert the token at the cursor — no memorizing `{date}`.
3. **Expanded vocabulary** — ~9 tokens (the above), locale-aware, expanded **at insert time**.
4. **Proper editor sheet** (title + body + **live preview** of the expanded result) replacing the inline cell.

**Touchpoints:** `SnippetsViewController.swift` (composer → sheet + chips + preview);
`SharedExtensions.swift` (`Snippet`, `SnippetsManager`, token-expansion — add the new tokens);
`Keyboard/Views/SnippetsListView.swift` (keyboard-side expansion must understand the new tokens).

## Section 3 — Custom Keys: action-sheet hell → one preview screen

**Problems:** editing the 3 right-edge slots means **3 separate action sheets**; "Custom…" pops a
*second* alert; no preview.

**Redesign (the professional UX pass):**
1. **Live keyboard preview at the top** showing the real right-edge keys; **tap a key in the preview**
   to edit it (direct manipulation, not abstract "Top Right Key" rows).
2. **Inline token palette** (chips: `, . Space Tab ← → Hide` + **Custom…**); one tap assigns, preview
   updates instantly; "Custom…" reveals an inline field (no separate alert).
3. **Build-your-own pack row** uses the same preview + inline-add + **springboard drag-to-reorder**
   (consistent with Section 4).

**Gating:** basic right-edge slot remapping **free**; build-your-own pack row **Pro**.

**Touchpoints:** `CustomKeysViewController.swift` (full rebuild around a preview + inline palette);
`SharedExtensions.swift` (`CustomKeys`, `CustomPackManager` — unchanged data model, new editing UX).

## Section 4 — Custom Keyboard: one keyboard, straight to setup, springboard drag-and-drop

### Navigation

- The **"Custom Keyboard"** row opens the **editor directly** for a single canonical layout — the
  list screen is skipped.
- `LayoutStore`'s multi-layout array **stays in code**, always pointing at one "primary" layout id.
  If none exists, seed from `KeyboardLayout.standard` on first open.
- This single-canvas model is the foundation a future **full-QWERTY** mode slots into (different
  seed + more columns).

### Springboard drag-and-drop (Approach B — hand-rolled)

Chosen over (A) the current `.draggable`/`.dropDestination` and (C) a third-party grid package
because it is the only one that delivers the home-screen-icon feel. The editor lives in the **app
target**, so the keyboard extension's memory limit does not apply.

Mechanics:
- **Lift:** long-press (~0.3s) → key scales up + shadow + haptic (optional gentle wiggle).
- **Drag:** key follows the finger; the grid computes the slot under it.
- **Reflow:** the other keys **animate to open a gap** at the insertion point (the signature move).
- **Drop:** snaps into the slot, haptic, commits.
- **Delete:** in edit mode, removable keys show a **⊖ badge**; **digits 0–9, delete, return are
  locked** (reorderable but not removable — preserves the existing `repaired()` guarantee).

### Internal model

The editor flattens the layout into a **single ordered list over a fixed 5-column grid** (clean
index math for reflow), then maps back to `rows` on save. Pro gating on save/activate stays as-is.

**Touchpoints:** `Customization/KeyboardLayout.swift`, `LayoutEditorModel.swift`, `LayoutEditing.swift`,
`LayoutStore.swift` (single-primary accessor), `Editor/LayoutListView.swift` (de-surface — keep code),
`Editor/LayoutGridEditorView.swift` (**rebuild** the gesture/reflow), `Editor/CustomKeyboardCoordinator.swift`
(open editor directly), `HomeViewController.swift` (row target).

## Section 5 — Notifications: reframe as updates, ask the right way

**Problem:** `EarlyBird.swift` calls `requestAuthorization` **directly** — a one-shot dialog fired
purely for promo pushes. A "Don't Allow" is permanent; App Review dislikes push-for-marketing.

**Redesign:**
1. **In-app pre-prompt before the system dialog** (the mitigation already flagged in the deferred log):

   > **Stay in the loop**
   > NumPad can send occasional notifications about **new features and important updates** — and the
   > rare early-bird offer. No spam, ever.
   > [ Not now ]   [ Turn on updates ]

   Only **"Turn on updates"** triggers the system dialog, so "Not now" never burns the one shot.
2. **Reframe positioning** — permission is about *updates*; the early-bird offer is *one such update*.
   The two reminder pushes stay under the "updates" umbrella.
3. **Ask contextually** — surface the pre-prompt once the user is engaged (e.g., after enabling the
   keyboard, or inside the early-bird flow), never on first paint.

**Touchpoints:** `EarlyBird.swift` (pre-prompt gate + timing); a small pre-prompt view/alert in the app target.

## Out of scope / future

- **Full-QWERTY keyboard** with custom-key options — a deliberate *later* build; this redesign lays
  the single-canvas foundation for it.
- **Multiple custom keyboards** — code retained, intentionally unsurfaced.
- **Live FX currency conversion** (Units pack) — already cut.

## Owner-gated (cannot be done in code)

- **ASC:** reduce to **6 IAPs** (4 packs + 2 Pro); pricing/approval; verify Finance grandfathering.
- **Device regression** on both phones (iOS 26.5 + 27.0), especially the springboard reflow gesture.
- **App Store submission** — owner clicks Submit.

## Build order (proposed)

1. **Packs 12 → 5** (foundation: gating + grandfathering everything else references).
2. **Notifications** (small, isolated; de-risks App Review early).
3. **Snippets** (richer tokens + composer).
4. **Custom Keys** (preview screen + inline palette).
5. **Custom Keyboard** (the big lift: direct-to-editor + springboard reflow).
