# NumPad 2.0 — Ship Checklist (owner runbook)

2.0 is built on `feat/2.0` (all green @ 117 tests, simulator-built). The **code + config are done**;
everything below is owner-gated (App Store Connect, signing, physical devices, submit). Product IDs
and prices match `NumPad/Products.storekit`. See `2026-06-24-2.0-deferred-log.md` for decisions to review.

## 1. App Store Connect — products & pricing
- Set the **app price tier to $2.99** (the app becomes PAID; existing downloaders keep it for free).
- **Reprice `numpad.pro.lifetime` → $11.99** (existing owners are unaffected).
- Create non-consumables **@ $1.99**: `numpad.pack.symbols`, `.programmer`, `.units`, `.scientific`,
  `.datetime`, `.business`, `.intl`, `.programmerplus`. (`numpad.pack.finance` already exists — keep @ $1.99.)
- Create non-consumable **`numpad.pro.lifetime.earlybird` @ ~$5.99** (early-bird 50%-off Pro).
- Add display name / description / review screenshot for each IAP; submit the IAPs with the build.

## 2. Capabilities / signing
- Enable **iCloud → Key-Value storage** for `com.morevoltage.NumPad` in the Developer portal
  (entitlement is already declared in `NumPad/NumPad.entitlements`); regenerate the App Store profile.
- Simulator testing of purchases: Edit Scheme → Run → Options → **StoreKit Configuration → `Products.storekit`**.

## 3. Version + metadata
- Bump **`MARKETING_VERSION` → 2.0** and **`CURRENT_PROJECT_VERSION`** (both targets) before archiving.
- **en-US** metadata is rewritten for 2.0 (description, release notes, promotional text). The other
  **49 locales still read 1.8.1** — regenerate translations before submit (pattern:
  `marketing/release_notes_1_8_0.py`, or the localization pipeline in the shabbat-localization workflow).
- Refresh screenshots if they show the old pack set / store.

## 4. Build & device verification (both phones: iPhone 16 Pro 26.5, iPhone 17 27.0)
- New packs render + insert (incl. Date & Time live values); Programmer+/International/etc.
- **Custom keyboard (v2, structured editor)**: build a top row + side columns (left/right-handed); packs cycle through the top row; type keys via the forced system keyboard. _(✅ Device-verified on both phones. Lives on `feat/2.0-ux-overhaul`, **not yet merged to `feat/2.0`**; the springboard editor it replaced is deleted. See `2026-06-24-custom-keyboard-v2-design.md` → As-built.)_
- GA behaviors: `=` calculates, long-press return tape, space-drag cursor, smart pack defaulting.
- Purchases via the `.storekit` config: buy a pack (unlocks just it), buy Pro (unlocks all + custom kbd + themes + sync), early-bird row appears for a grandfathered test profile.
- **iCloud sync** across two iCloud-signed devices (snippets/layouts/packs/settings propagate). _(Never device-verified.)_
- Archive + upload via the manual-signing recipe (see `numpad-status` memory / `marketing/asc/release-1.8.0-runbook.md`).

## 5. Submit
- Attach the build + IAPs; set "Sign-in not required"; export compliance = no non-exempt encryption.
- **A human clicks Submit — never auto-submitted.**

## What is NOT done in code (deferred — see deferred log)
- Clipboard is excluded from iCloud sync (keychain/security); proper path is iCloud Keychain.
- Early-bird notification-permission timing is an App Review risk (contextual pre-prompt recommended).
- iCloud conflict strategy is last-writer-wins per key (no union merge).
- Pack key contents are first-pass; inline calculator was promoted to GA alongside the result tape.
- Custom keyboard v2 follow-ups: editor geometry/UX tuning, and re-surfacing Custom Keys as custom packs + macros (deferred log). The old springboard follow-ups (per-key colour/size, label override, export/import, advanced tokens) are obsolete — that editor was deleted.
