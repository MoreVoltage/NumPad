## Monetization model (LOCKED)

### Base — $2.99 paid download
- Packs: `default` + `math` + `math2`. Free (non-premium) themes.
- **All free UX features stay free** (they're features, not packs): calculator, cursor
  controls, smart pack defaults, result/history tape, clipboard/snippets/tax-tip overlays,
  locale-aware separators, fast-delete.

### À la carte packs — $1.99 each (non-consumable)
Nine packs:
- **Originals, now sold individually:** Finance (`numpad.pack.finance`, exists), Symbols, Programmer.
- **New (6):** Units&Conversion, Scientific, Date/Time, Business/Accounting, International/Formatting, Programmer-Extended.

> ASSUMPTION (confirm-by-omission, see deferred log): the original premium packs go à la carte
> at $1.99. Owner did not object when flagged. Same gating mechanism either way — trivial to
> flip any pack to Pro-only later.

### Pro — $11.99 (non-consumable, one IAP)
Everything: all 9 packs **+ Customizable Keyboard (Pro-EXCLUSIVE — the only way to get it)**
+ premium themes + custom-keys row + iCloud sync + all future packs.

### Grandfathering
- Existing **$4.99 Pro** owners → keep Pro (auto-upgraded to the richer Pro). ✅
- Existing **Finance Pack** owners → finance stays unlocked. Finance remains on sale at $1.99
  for everyone else (NOT retired).
- Existing **non-Pro** users (free + finance-only) → keep base; eligible for the early-bird offer.
- New **$2.99** buyers → base.
- **Detection:** reuse existing grandfather plumbing — `AppTransaction.originalAppVersion`,
  production-only guard (sandbox always reports "1.0"), `grandfatherCheckedV2` one-shot — plus
  StoreKit `Transaction.currentEntitlements`. Add a 2.0 migration one-shot flag.

### Early-bird Pro promo (LOCKED)
- **Eligibility:** pre-2.0 / grandfathered user who does NOT already own Pro, within **72 hours**
  of first 2.0 launch.
- **Product:** new non-consumable `numpad.pro.lifetime.earlybird` (~$5.99, 50% off) → grants the
  identical Pro entitlement. (Native StoreKit discounts are subscription-only, so a second SKU is
  the only way to discount a non-consumable per-user.)
- **Window:** 72h from a persisted first-2.0-launch timestamp (app group). After expiry → only
  full-price Pro is shown.
- **Local notifications (2):** N1 at `start + 1h` ("your 50%-off Pro is live"); N2 at
  `start + 66h` (= 72h − 6h, "last chance — ends in 6 hours"). Request notification auth
  **contextually** when the offer is first surfaced (pre-prompt); if granted after N1's time,
  schedule only the still-future ones. **Cancel both** on Pro purchase (either SKU) or window expiry.
- **Review/UX risk** (deferred log): requesting notification permission to advertise an offer is
  borderline per App Review — mitigate with a contextual pre-prompt; the in-app countdown works
  with or without permission.

## iCloud sync (Pro feature)
CloudKit runs **app-side only** (cannot run in the keyboard extension) and is mirrored to the
extension via the shared app group + `SettingsSync` Darwin notifications. Syncs (owner option 2):
layouts, snippets, custom-pack keys, settings/preferences, **and clipboard history**.

## Per-phase build approach

- **Phase 2 — packs.** Per pack: engine (TDD in `/tmp` SPM, port into
  `NumPad/Libraries/.../` + `NumPadTests/`) → `Item.pack(type:)` layout → `KeyboardType` case +
  `.packs` → gating + product id. Reuse `UnitConverter` (extend units), `Calculator`,
  `TaxTipMath`. New engines: scientific, date/time arithmetic, business/finance math, intl
  formatting (tight — no libphonenumber), programmer-extended (binary/ASCII). **Currency
  conversion needs live FX rates → deferred.**
- **Phase 3 (subset) — promote + build.** Flip `cursorControls`, `smartPackDefaulting`,
  `lastResultTape` from experimental flag to GA (default-on, de-gate, polish, test). **Build
  iCloud sync** (CloudKit container + record types + merge + app-group mirror).
- **Phase 4 — Features & Guide.** In-app help screen (packs, custom keyboard, overlays, Pro,
  enabling Full Access). Likely a UIKit table or a small SwiftUI island matching existing screens.
- **Phase 5 — monetization restructure.** Product IDs; **local `.storekit` config** for sim
  testing; paid-app gating defaults; v1→v2 migration; early-bird (window + notifications +
  cancel); Store UI rewrite for the new ladder; **remove the standalone-$4.99 custom-kbd path**;
  retire/repurpose the first-run paywall funnel (paid app → no aggressive upsell; keep contextual).
- **Phase 6 — ship prep.** ASC metadata rewrite (50 locales); finalize `.storekit`; archive
  build; write the owner checklist. **No submission, no device regression** (owner-gated).
