# Keyword Update Plan

Goal: tighten the locale keyword sets around high-intent numeric-entry use cases, while keeping the current translations intact unless there is a clear reason to change them.

## Priority Order

1. English master copy.
2. Spanish (`es-ES`).
3. Simplified Chinese (`zh-Hans`).
4. Traditional Chinese (`zh-Hant`).
5. Hebrew (`he`).
6. Arabic (`ar-SA`).
7. The remaining locales, only after the highest-volume markets are stable.

## English Master

Recommended keyword cluster:

`numpad,number pad,number keyboard,numeric keypad,calculator keyboard,spreadsheet keyboard,forms keyboard,clipboard history,tax tip,finance keyboard,programmer keyboard,pin,otp`

Why:

- Search is already the biggest product-page source.
- These terms line up with the app's actual workflows: forms, spreadsheets, finance, clipboard, tax/tip, and programmer use.

## Spanish

Recommended keyword cluster:

`teclado numerico,teclado de numeros,formularios,hojas de calculo,excel,portapapeles,finanzas,impuesto,propina,pin,otp,codigo`

Why:

- The existing Spanish metadata already supports numeric and finance language.
- Spanish is efficient coverage for Spain and many Latin markets.

## Simplified Chinese

Recommended keyword cluster:

`数字键盘,数字,键盘,表格,Excel,表单,验证码,PIN,财务,税费,小费,符号,代码`

Why:

- Chinese metadata already uses a strong numeric-entry vocabulary.
- The app's real use cases map cleanly to forms, Excel, verification codes, and finance.

## Traditional Chinese

Recommended keyword cluster:

`數字鍵盤,數字,鍵盤,表格,Excel,表單,驗證碼,PIN,財務,稅費,小費,符號,代碼`

Why:

- Keep the same intent as Simplified Chinese, but write in native traditional characters.

## Hebrew

Recommended keyword cluster:

`מקלדת מספרים,מספרים,טפסים,אקסל,פיננסים,מס,טיפ,סמלים,קוד,PIN,OTP,כרטיס`

Why:

- Hebrew is already localized and the keyboard utility story is strong.
- Keep the terms native and concise.

## Update Rules

- Do not expand the keyword list just because space is available.
- Prefer intent density over synonyms.
- Keep title and subtitle changes separate from keyword changes unless there is a clear testing reason to move them together.
- Validate each locale for spacing, separators, and script direction before upload.
- Leave long-tail locales alone until the highest-return locales have been tested.
