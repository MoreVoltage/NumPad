# Custom Product Pages Plan

Goal: create page variants that match the user's intent instead of sending every visitor to the same generic storefront.

## Proposed CPPs

| CPP Slug | Audience | Core Message | Best Asset Set |
|---|---|---|---|
| `numpad-workflow` | General productivity and search traffic | Numbers without slowdowns, faster forms, paste recent numbers | Screenshots 1, 2, 4 |
| `numpad-finance` | Finance, tax, and spreadsheet users | Tax and tip plus finance symbols | Screenshots 2, 3, 5 |
| `numpad-power` | Power users and customizers | Pro packs, clipboard history, and style | Screenshots 4, 5, 6 |

## Mapping Ideas

- Use `numpad-workflow` for broad App Store Search traffic.
- Use `numpad-finance` for finance-intent search terms and Apple Ads groups.
- Use `numpad-power` for re-engagement, returning users, and premium-feature discovery.

## Measurement Plan

- Compare each CPP against the control product page.
- Track conversion rate, product page views, and first-time downloads.
- Watch whether the finance CPP increases IAP attach rate without hurting overall conversion.
- Use the winning message structure as the source for future localized CPPs.

## Asset Rules

- Keep the same visual system as the main page so the CPP feels like a focused variant, not a separate brand.
- Reuse the existing screenshot concepts instead of inventing new ones for the first round.
- Localize only the CPPs that match the highest-volume markets first.

## Recommendation

Start with `numpad-workflow` because it best matches the current search-led acquisition mix.
